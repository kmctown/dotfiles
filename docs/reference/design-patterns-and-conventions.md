# Design Patterns and Conventions

## Topic-Centric Organization

Every domain gets its own top-level directory. No nesting beyond `topic/file`. Adding a new topic is: create the directory, add files with the right names, done.

## File Naming Contract

| Name | Behavior | Loaded By |
|---|---|---|
| `path.zsh` | PATH modifications | `zshrc.symlink` (first pass) |
| `*.zsh` | Shell config, aliases, functions | `zshrc.symlink` (second pass) |
| `completion.zsh` | Autocomplete setup | `zshrc.symlink` (third pass, after compinit) |
| `*.symlink` | Symlinked to `$HOME/.{name}` | `script/bootstrap` |
| `install.sh` | Optional installer | `script/install` |

## Shell Conventions

**Aliases**: defined as `alias name='command'` in `topic/aliases.zsh`. Short, mnemonic names (`gs`, `gl`, `gp`, `gd`).

**Functions**: defined with `function name() { ... }` in `topic/aliases.zsh` or `functions/`. Use functions when logic is needed (conditionals, guards). Use aliases for simple command shortcuts.

**Guard clauses**: always check for optional tools before using them:
```sh
# Preferred patterns:
if command -v tool &> /dev/null; then ...
(( $+commands[tool] )) && ...
```

**Exit codes**: `install.sh` scripts exit 0 on success. Use `set -e` only in bootstrap (not in individual installers).

## Install Script Pattern

Every `install.sh` follows:
1. Check if the tool is already installed
2. Install only if missing
3. Create config directories if needed
4. Symlink config files
5. Exit 0

```sh
#!/bin/sh
if ! command -v tool &> /dev/null; then
    echo "  Installing tool"
    brew install tool
fi
# ... symlink config ...
exit 0
```

## Symlink Safety

`script/bootstrap` handles conflicts interactively (skip, overwrite, backup). It re-runs symlinking after all installers complete because oh-my-zsh's installer overwrites `~/.zshrc`.

## Secrets Management

- `~/.localrc` — private env vars, API keys (sourced in zshrc, gitignored)
- `git/gitconfig.local.symlink` — local git identity (generated by bootstrap, gitignored)
- Never commit `.env`, credentials, or API keys to topic files

## Error Handling

- Optional tools use guard clauses — missing tools don't break shell startup
- `brew bundle` uses `|| true` to continue if a cask is unavailable
- `script/bootstrap` uses `set -e` for the critical setup path
- Warning messages use colored printf: `printf '\033[1;33m[warning]\033[0m\n'`

## Git Conventions

- Commit messages: imperative, prefixed (`feat:`, `fix:`, `chore:`)
- Pull strategy: rebase (`pull.rebase = true`)
- Default branch: `main` (for new repos)
- Credential helper: `gh auth git-credential`
- Custom git commands in `bin/` (e.g., `git-nuke`, `git-delete-local-merged`)

## macOS Conventions

- System defaults applied via `defaults write` in `macos/set-defaults.sh`
- Hostname set via `macos/set-hostname.sh`
- Apple Silicon detection: `test "$(uname -m)" = "arm64"`
- Homebrew path: `/opt/homebrew` (Apple Silicon) or `/usr/local` (Intel)

## Maintenance & Accretion

Update this doc when new patterns emerge or existing conventions are deprecated. If a convention is violated in practice, either fix the code or update this doc — don't let them drift apart.
