# Git Configuration

## Purpose

Git config, shell aliases, custom commands, and credential management. Two-layer config: shared (`gitconfig.symlink`) and local (`gitconfig.local.symlink`).

## How It Works

### Key Files

| File | Purpose |
|---|---|
| `git/gitconfig.symlink` | Shared git config → `~/.gitconfig` |
| `git/gitconfig.local.symlink` | Local identity (name, email) → `~/.gitconfig.local` (gitignored) |
| `git/gitignore.symlink` | Global gitignore → `~/.gitignore` |
| `git/aliases.zsh` | Shell aliases for common git operations |
| `git/completion.zsh` | Git command completion |

### Config Hierarchy

```
~/.gitconfig (shared)
  ├── [include] path = ~/.gitconfig.local  (local identity)
  ├── [alias] promote, wtf, rank-contributors (call bin/ scripts)
  ├── [credential] helper = gh auth git-credential
  ├── [pull] rebase = true
  └── [init] defaultBranch = main
```

### Credential Helper

Uses GitHub CLI: `helper = !/opt/homebrew/bin/gh auth git-credential`. Requires `gh auth login` to be run once.

## Interfaces, APIs, and Contracts

### Shell Aliases (`git/aliases.zsh`)

| Alias | Command |
|---|---|
| `gs` | `git status -sb` |
| `gl` | `git pull --prune` |
| `gp` | `git push origin HEAD` |
| `gd` | `git diff` (colored) |
| `gdm` | `git diff main...HEAD` |
| `gac` | `git add -A && git commit -m` |
| `gc` | `git commit` |
| `gca` | `git commit -a` |
| `gco` | `git checkout` |
| `gcb` | `git copy-branch-name` |
| `gb` | `git branch` |

### Custom Git Commands (`bin/`)

| Command | Purpose |
|---|---|
| `git-nuke <branch>` | Delete branch locally + remotely |
| `git-delete-local-merged` | Clean up merged branches |
| `git-promote` | Promote branch to tracking |
| `git-rank-contributors` | Rank contributors by commit count |
| `git-wtf` | Show branch status |
| `git-up` | Pull with rebase |
| `git-undo` | Soft reset last commit |
| `git-amend` | Amend last commit (same message) |
| `git-unpushed` | Show unpushed commits |

## Design Patterns and Conventions Used Here

- **Two-layer config**: Shared config committed to repo; local identity generated by bootstrap and gitignored
- **Shell aliases + git aliases**: Shell aliases (`gs`, `gl`) for daily use; git aliases (`promote`, `wtf`) for less common operations that call bin/ scripts
- **gh CLI integration**: Credential helper avoids storing tokens in config files

## Code Examples and Gotchas

**First-time setup** (handled by `script/bootstrap`):
```sh
# Bootstrap prompts for these and creates git/gitconfig.local.symlink:
git config --local user.name "Your Name"
git config --local user.email "your@email.com"
```

**Gotcha**: `git/gitconfig.local.symlink` is gitignored. If you clone to a new machine, bootstrap will prompt for your identity again.

**Gotcha**: The credential helper path is hardcoded to `/opt/homebrew/bin/gh` (Apple Silicon). On Intel Macs, this path would be `/usr/local/bin/gh`.

## Maintenance & Accretion

Update this doc when git aliases are added/removed or the credential helper changes. The alias table should match `git/aliases.zsh`.
